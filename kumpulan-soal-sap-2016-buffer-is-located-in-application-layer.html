1. apakah pengertian dan fungsi dari application layer dan contoh dari application layer ??


Secara singkat, Application layer merupakan salah satu layer pada Open System Interconnection (OSI), fungsi layer ini sendiri adalah menyediakan tampilan (interface) kepada pengguna, contoh fungsi dari application layer ini diantaranya:
- File Transfer
- Web Surfing
- Web Chat
- Simple Mail Transfer Protocol (SMTP)

25. protokol-protokol yang ada di : 1. Phisycal layer 2. Data Link layer 3. Network layer 4. Transport layer 5. Session layer 6. Presentation layer 7. Application layer


mungkin maksud anda tugas  dari setiap layer tersebut.

- Physical Layer memiliki fungsi mengirimkan raw bit ke chanel komunikasi.
- Tugas utama dari data link layer adalah  sebagai fasilitas transmisi raw data dan mentransformasi data tersebut ke saluran yang bebas dari kesalahan transmisi.
- network layer berfungsi untuk pengendalian operasi subnet.
- Fungsi dasar transport layer adalah menerima data dari atau ke session layer, memecah data menjadi bagian-bagian yang lebih kecil, dan meneruskannya ke network layer
- session layer menyediakan layanan-layanan yang istimewa untuk aplikasi-aplikasi tertentu

- Presentation layer melakukan fungsi-fungsi tertentu yang diminta untuk menjamin masalah-masalah umum pada data.

- pada aplication layer ada bermacam-macam protokol untuk menjalankan tugas yang dimiliki masing-masing protocol yang bekerja dengannya.

26. Jelaskan perbedaan antara Application Layer dan Transport Layer serta masing-masing peranannya dalam komunikasi data!​

Application layer, sebagai pelepas data dalam sebuah jaringan dan juga penampil data dalam sebuah jaringan memiliki beberapa fungsi, seperti : Menyajikan interface antara aplikasi dengan jaringan. Mengatur bagaimana sebuah aplikasi mampu untuk mengakses jaringan. Membuat pesan – pesan berupa kesalahan pada jaringan

Transport Layer Adalah

Dalam model OSI, lapisan ini secara umum bertugas untuk menggabungkan data yang diambil dari banyak sumber untuk dikirimkan dalam satu jaringan saja.

Lapisan transportasi memiliki tugas memecah data, memberi penomoran pada tiap paket data, dan menjalankan transmisi data ulang apabila terdeteksi adanya paket yang hilang.

HTTP (Protokol Transfer Hiperteks)

HTTP adalah protokol yang digunakan untuk mentransfer dokumen di browser web World Wide Web, yang berfungsi untuk merespons antara klien dan server, dan untuk membuat koneksi TCP/IP ke port tertentu pada host jarak jauh. Server HTTP yang mendengarkan pada port itu menunggu klien mengirim kode permintaan.

Port HTTP: 80

FTP (Protokol Transfer File)

FTP adalah protokol Internet yang merupakan standar untuk mentransfer file komputer antar mesin dalam sebuah internetwork. FTP digunakan untuk mengunduh dan mengunggah file komputer antara klien FTP dan server FTP. Perintah FTP dapat digunakan untuk mengubah direktori, mengubah mode transfer antara biner dan ASCII, mengunggah file komputer ke server FTP, dan mengunduh file dari server FTP.

Port: port TCP 20 dan 21

Protokol Transfer Surat Sederhana (SMTP)

SMTP (Simple Mail Transfer Protocol) adalah salah satu protokol yang biasa digunakan untuk pengiriman surat elektronik (e-mail) di Internet. SMTP berfungsi untuk mengirim pesan email dari klien email ke server email, mengirim email ke akun lokal, dan menyiarkan email antar server SMTP.

Pada dasarnya email yang dikirim oleh server SMTP

akan ditampung sementara oleh server komputer untuk dikirim ke server komputer tujuan. Komputer server pengirim dan komputer server penerima berinteraksi pada port 25 untuk mengirim email.

Port: port TCP 25

Contoh Layanan: E-Mail

Sistem File Jaringan (NFS)

NFS umumnya menggunakan protokol Remote Procedure Call (RPC) yang berjalan di atas UDP dan membuka port UDP dengan nomor port 2049 untuk komunikasi antara klien dan server di jaringan. Klien NFS kemudian mengimpor sistem file jarak jauh dari server NFS, sedangkan server NFS mengekspor sistem file lokal ke klien. Mesin yang menjalankan perangkat lunak klien NFS dapat berinteraksi dengan perangkat lunak server NFS untuk melakukan operasi perintah tertentu menggunakan permintaan RPC. Operasi yang didukung oleh NFS adalah sebagai berikut:

· Cari file dalam direktori.

· Membaca kumpulan direktori.

· Memanipulasi link dan direktori.

· Akses atribut file.

· Membaca dan menulis file.

Perlu diketahui bahwa server NFS bersifat stateless, artinya setiap permintaan harus berisi argumen yang lengkap dan jelas karena server NFS tidak menyimpan riwayat informasi permintaan. Data yang dimodifikasi harus dikomit ke server sebelum hasilnya dikembalikan ke klien. Protokol NFS tidak menyediakan mekanisme kontrol konkurensi.

           (Port) : UDP 2049
